**Scenario:** Your organization utilizes AWS to host critical data and applications. An incident has been reported that involves unauthorized access to data and potential exfiltration. The security team has detected unusual activities and needs to investigate the incident to determine the scope of the attack.

## Questions

> Q1: Knowing which user account was compromised is essential for understanding the attacker's initial entry point into the environment. What is the username of the compromised user?

![](assets/cyberdef/Pasted%20image%2020260606181925.png)

Firstly, we can enumerate what AWS `eventSource`available in the environment. We can look into event source`signin.amazonaws.com` to identify any suspicious login.

![](assets/cyberdef/Pasted%20image%2020260606182324.png)

![](assets/cyberdef/Pasted%20image%2020260606182444.png)

![](assets/cyberdef/Pasted%20image%2020260606182926.png)

To identify suspicious login, I filter failed logon and noticed that `helpdesk.luke`has high amount of login failure. There are multiple failed login attempts followed by a successful logon.

![](assets/cyberdef/Pasted%20image%2020260606183524.png)

After the successful logo, there are high volume of enumeration activities (List*, Describe*, Get*) executed by the `helpdesk.luke`。


>✅ **Answer: helpdesk.luke**

---

> Q2: We must investigate the events following the initial compromise to understand the attacker's motives. What is the timestamp for the first access to an S3 object by the attacker?

![](assets/cyberdef/Pasted%20image%2020260606185408.png)

The successful logon was at 2023-11-02 09:54:04. We can filter the time and event source `earliest=11/02/2023:09:54:04 eventSource="s3.amazonaws.com"` and find the `eventName` that relate to S3. Usually, attacker would execute `ListBuckets`to enumerate what is inside the bucket.

>✅ **Answer: 2023-11-02 09:55**

---

> Q3: Among the S3 buckets accessed by the attacker, one contains a DWG file. What is the name of this bucket?

![](assets/cyberdef/Pasted%20image%2020260606185854.png)

To find the S3 bucket containing DWG file, we can add the parameter `*dwg*` in the query. There are only 3 events with the dwg in the data. Search or filter  `eventName: GetObject` because this api means retrieves an object from Amazon S3.

![](assets/cyberdef/Pasted%20image%2020260606190435.png)

The bucket name is the portion immediately following `s3://` .

>✅ **Answer: product-designs-repository31183937**

---

> Q4: We've identified changes to a bucket's configuration that allowed public access, a significant security concern. What is the name of this particular S3 bucket?

![](assets/cyberdef/Pasted%20image%2020260606191722.png)

![](assets/cyberdef/Pasted%20image%2020260606191727.png)

Filter `PutBucketPublicAccessBlock` to query any S3 public access configuration made to the bucket.

>✅ **Answer: backup-and-restore98825501**

---

> Q6: Creating a new user account is a common tactic attackers use to establish persistence in a compromised environment. What is the username of the account created by the attacker?

![](assets/cyberdef/Pasted%20image%2020260606191926.png)

![](assets/cyberdef/Pasted%20image%2020260606192013.png)

Filter `eventName="CreateUser"` to find user created.

>✅ **Answer: marketing.mark**

---

> Q6: Following account creation, the attacker added the account to a specific group. What is the name of the group to which the account was added?

![](assets/cyberdef/Pasted%20image%2020260606192259.png)

![](assets/cyberdef/Pasted%20image%2020260606192320.png)

Filter `eventName="AddUserToGroup"`and search for `marketing.mark`. The account created by the attacker was added the Admins group.

>✅ **Answer: Admins**
