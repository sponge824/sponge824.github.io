**Scenario:** After Karen started working for 'TAAUSAI,' she began doing illegal activities inside the company. 'TAAUSAI' hired you as a soc analyst to kick off an investigation on this case.

You acquired a disk image and found that Karen uses Linux OS on her machine. Analyze the disk image of Karen's computer and answer the provided questions.

## Questions

In this lab, I was provided with a disk image file, FirstHack.ad1. To analyze the Linux machine, I mounted the disk image in FTK Imager by navigating to File → Add Evidence Item → Image File, and then select the image file

> Q1: Which Linux distribution is being used on this machine?

![](assets/cyberdef/Pasted%20image%2020260530165011.png)

The linux distribution can be found in boot directory. It is running kali linux.

>✅ **Answer: kali

---

> Q2: What is the MD5 hash of the Apache **access.log** file?

![](assets/cyberdef/Pasted%20image%2020260530170143.png)

`access.log` can be located in `/var/log/apache2`. It records every HTTP request made to the web server. To get the MD5 hash, right click -> Export File Hash List.

>✅ **Answer: d41d8cd98f00b204e9800998ecf8427e**

---

> Q3: It is suspected that a credential dumping tool was downloaded. What is the name of the downloaded file?

![](assets/cyberdef/Pasted%20image%2020260530170436.png)

There is a file downloaded under `/root/Downloads`. Mimikatz is a common tool used by attacker to dump hashes which are valuables for privilege escalation and lateral movement.

>✅ **Answer: mimikatz_trunk.zip**

---

> Q4: A super-secret file was created. What is the absolute path to this file?

![](assets/cyberdef/Pasted%20image%2020260530171740.png)

I could not locate the super-secret file. It may have been deleted by the attacker. To investigate further, I examined the `.bash_history` file, which records commands executed by the root user. The command history indicates that the super-secret file was originally created in `/root/Desktop`.

>✅ **Answer: /root/Desktop/SuperSecretFile.txt**

---

> Q5: What program used the file **didyouthinkwedmakeiteasy.jpg** during its execution?

Further analysis of `.bash_history` revealed that binwalk was executed against the file. Binwalk is a forensic and reverse engineering tool used to identify and extract embedded files and data from image files and other binary formats.

![](assets/cyberdef/Pasted%20image%2020260530172309.png)

>✅ **Answer: binwalk**

---

> Q6: What is the third goal from the checklist Karen created?

The checklist was located at `/root/Desktop`. The third goal created by Karen is Profit

![](assets/cyberdef/Pasted%20image%2020260530172615.png)

>✅ **Answer: Profit**

> Q7: How many times was Apache run?

The logs in `/var/log/apache2` revealed that Apache had never been run, as the log files in the directory were empty.

![](assets/cyberdef/Pasted%20image%2020260530173220.png)
