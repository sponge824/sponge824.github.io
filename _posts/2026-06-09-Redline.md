**Scenario:** As a member of the Security Blue team, your assignment is to analyze a memory dump using Redline and Volatility tools. Your goal is to trace the steps taken by the attacker on the compromised machine and determine how they managed to bypass the Network Intrusion Detection System (NIDS). Your investigation will identify the specific malware family employed in the attack and its characteristics. Additionally, your task is to identify and mitigate any traces or footprints left by the attacker.

## Questions

> Q1: What is the name of the suspicious process?

![](assets/cyberdef/Pasted%20image%2020260609174133.png)

![](assets/cyberdef/Pasted%20image%2020260609174525.png)

To list the processes present in the memory dump, plugin `windows.pslist` was utilized. After further analysis, a process `oneetx.exe`caught my attention because it did not appear to be legitimate Windows process. I conducted OSINT on the process, and the results indicated that is it a malware.

>✅ **Answer: oneetx.exe**

---

> Q2: What is the child process name of the suspicious process?

![](assets/cyberdef/Pasted%20image%2020260609183034.png)

By using `windows.pstree` plugin, the suspicious process spawned a `rundll32.exe`child process.

>✅ **Answer: rundll32.exe**

---

> Q3: What is the memory protection applied to the suspicious process memory region?

![](assets/cyberdef/Pasted%20image%2020260609184331.png)

The plugin `windows.malfind`revealed the memory protection applied to the suspicious process memory region is `PAGE_EXECUTE_READWRITE`.

>✅ **Answer: PAGE_EXECUTE_READWRITE**

---

> Q4: What is the name of the process responsible for the VPN connection?

![](assets/cyberdef/Pasted%20image%2020260609190830.png)

By looking back at the `windows.pslist`, I noticed the process `tun2socks.exe`, which is used to route network traffic. Its parent process is `Outline.exe` which is a VPN software.

>✅ **Answer: Outline.exe**

---

> Q5: What is the attacker's IP address?

![](assets/cyberdef/Pasted%20image%2020260609191444.png)

To identify the attacker's IP address, we can use plugin `windows.netscan` to list all the network connections. We can see that there is a connection from our internal IP to `77.91.124.20:80` which suggest a C2 communication.

>✅ **Answer: 77.91.124.20**

---

> Q6: What is the full URL of the PHP file that the attacker visited?

![](assets/cyberdef/Pasted%20image%2020260609193838.png)

![](assets/cyberdef/Pasted%20image%2020260609193939.png)

To track the attacker's activities, we can use `strings`command on the memory dump and output it to a `txt`file. Then, we can search the attacker's IP address `77.91.124.20` and look for php file.

>✅ **Answer: `http://77.91.124.20/store/game/index.php`**
