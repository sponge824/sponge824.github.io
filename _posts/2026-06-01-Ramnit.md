**Scenario:** Our intrusion detection system has alerted us to suspicious behavior on a workstation, pointing to a likely malware intrusion. A memory dump of this system has been taken for analysis. Your task is to analyze this dump, trace the malware’s actions, and report key findings.

## Questions

> Q1: What is the name of the process responsible for the suspicious activity?

![](assets/cyberdef/Pasted%20image%2020260601181120.png)

In this lab, I will be using Volatility3 to analyze the memory dump file provided.

By using `ps.tree` plugin in volatility3, most of the processes are legitimate windows processes. However, there is one suspicious process that caught my attention which is `ChromeSetup.exe`. This process probably trying to mimic legitimate Chrome Installation File


>✅ **Answer: ChromeSetup.exe**

> Q3: Identifying network connections is crucial for understanding the malware's communication strategy. What IP address did the malware attempt to connect to?

![](assets/cyberdef/Pasted%20image%2020260601182727.png)

To analyze the network connection of the memory dump, `windows.netscan`plugin was used. This plugin will shows the active and potentially terminated TCP/UDP connections

Based on the result, `ChromeSetup.ex`attempted to connect to `58.64.204.181:5202`. This is not a common port like (80,443,139,445) which increased the suspicion. 

>✅ **Answer: 58.64.204.181*

> Q5: Hashes serve as unique identifiers for files, assisting in the detection of similar threats across different machines. What is the SHA1 hash of the malware executable?

To obtain the file hash of the malware, we first need to identify the virtual address of the malicious file `vol3 -f memory.dump windows.dumpfile`. 

![](assets/cyberdef/Pasted%20image%2020260601214101.png)

Next, we can use plugin `windows.dumpfiles` and filter the virtual address to dump the cached file contents from the memory.

`vol3 -f memory.dmp windows.dumpfiles --virtaddr 0xca82b85325a0`

![](assets/cyberdef/Pasted%20image%2020260601185736.png)

Lastly, use `Get-FileHash <target-file> -Algorithms SHA1` to obtain SHA1 hash

![](assets/cyberdef/Pasted%20image%2020260601185642.png)

>✅ **Answer: 280C9D36039F9432433893DEE6126D72B9112AD2**
