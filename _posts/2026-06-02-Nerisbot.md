**Scenario:** Unusual network activity has been detected within a university environment, indicating potential malicious intent. These anomalies, observed six hours ago, suggest the presence of command and control (C2) communications and other harmful behaviors within the network.

## Questions

> Q1: During the investigation of network traffic, unusual patterns of activity were observed in Suricata logs, suggesting potential unauthorized access. One external IP address initiated access attempts and was later seen downloading a suspicious executable file. This activity strongly indicates the origin of the attack.  
> What is the IP address from which the initial unauthorized access originated?

![](assets/cyberdef/Pasted%20image%2020260602153154.png)

![](assets/cyberdef/Pasted%20image%2020260602155402.png)

Before diving in, we analyzed the available log sources. Within these logs, we are going to analyze Suricata as it is network IDS/IPS. We are going to focus on event_type `http` as mentioned it downloaded a suspicious executable file.

Then, we group the alerts based on the destination IP with the query below:

```
index=* sourcetype=suricata event_type=http
| stats values(src_ip) values(http.http_user_agent) values(http.url) by dest_ip
```

After grouping the alerts, there is only 1 IP stands out which is `195.88.191.59` where our server downloaded multiple `.exe`files.

![](assets/cyberdef/Pasted%20image%2020260602162601.png)


>✅ **Answer: 195.88.191.59**

---

> Q2: Investigating the attacker’s domain helps identify the infrastructure used for the attack, assess its connections to other threats, and take measures to mitigate future attacks. What is the domain name of the attacker server?

![](assets/cyberdef/Pasted%20image%2020260602161959.png)

In order to find the domain name, we are going to filter zeek's dns log

```
index=* sourcetype="zeek:dns" 195.88.191.59
```

The IP `192.88.191.59`resolved to `nocomcom.com`.

>✅ **Answer: nocomcom.com**

> Q5: What is the SHA256 hash of the malicious file disguised as a `.txt` file?

![](assets/cyberdef/Pasted%20image%2020260602171513.png)

![](assets/cyberdef/Pasted%20image%2020260602171541.png)

To identify which files is the `.txt`file, we can check the size of the file in suricata and compare it with Zeek's file logs. The size of the txt file is 37376 which maches the SHA1 hash `2a6d5ad9a782c96f9cd214fcd105056248e6df31`. However, the SHA256 hash is unavailable in this log.

![](assets/cyberdef/Pasted%20image%2020260602171802.png)

We can upload the SHA1 hash to VirusTotal and check the SHA256 hash of the `.txt`file.

>✅ **Answer: 6fbc4d506f4d4e0a64ca09fd826408d3103c1a258c370553583a07a4cb9a6530**
