## Scenario

During your shift as a tier-2 SOC analyst, you receive an escalation from a tier-1 analyst regarding a public-facing server. This server has been flagged for making outbound connections to multiple suspicious IPs. In response, you initiate the standard incident response protocol, which includes isolating the server from the network to prevent potential lateral movement or data exfiltration and obtaining a packet capture from the NSM utility for analysis. Your task is to analyze the pcap and assess for signs of malicious activity.

## Methodology

### Initial Access

![](assets/cyberdef/Pasted%20image%2020260623120233.png)

![](assets/cyberdef/Pasted%20image%2020260623121734.png)

![](assets/cyberdef/Pasted%20image%2020260623121757.png)

I started by analyzing the conversations statistics in Wireshark to get an overview of the traffic. Since, `134.209.197.3`appeared multiple times, this is likely our public-facing server. I noticed a large volume of traffic between `146.190.21.92` and `134.209.197.3`, which stood out because it is unusual to see a high volume of traffic between an external IP and our server. To investigate, I filtered the IP with `ip.addr == 146.190.21.92`to identify abnormal packets. The result showed there's a suspicious `GET`request, by following the HTTP stream it revealed that our web server (`134.209.197.3`) retrieved `/invoice.xml` from `146.190.21.92`. Inside the `/invoice.xml`, it trying to download a file called `docker`from `128.199.52.72` and saves it in local machine `/tmp/docker`, followed by makes the downloaded file executable and attempts to execute it.

![](assets/cyberdef/Pasted%20image%2020260623123914.png)
				*Source: [Apache ActiveMQ Exploit Leads to LockBit Ransomware](https://thedfirreport.com/2026/02/23/apache-activemq-exploit-leads-to-lockbit-ransomware/)*


![](assets/cyberdef/Pasted%20image%2020260623123641.png)

![](assets/cyberdef/Pasted%20image%2020260623123659.png)

However, the initial access point still remains unclear. To investigate, I traced back the previous packet and I noticed that the malicious IP `146.190.21.92` send a maliciously crafted OpenWire command to our ActiveMQ server which lead to RCE. 

![](assets/cyberdef/Pasted%20image%2020260624132447.png)

![](assets/cyberdef/Pasted%20image%2020260624132519.png)

Apart from the identified IP, I noticed another suspicious IP `128.199.52.72`, which stood out because our web server retrieved a `docker`file from it. Upon further analysis of the downloaded file, it appears to be a `.elf` (Linux executable) which is commonly used to establish a reverse shell. Reverse shell allows a compromised machine to connect back to an attacker's listening host, the threat actor can execute command on the compromised system remotely.

![](assets/cyberdef/Pasted%20image%2020260624133429.png)

![](assets/cyberdef/Pasted%20image%2020260624133454.png)

By obtaining the hash and upload it to Virustotal, we can confirm that this is a reverse shell in `elf`format.
