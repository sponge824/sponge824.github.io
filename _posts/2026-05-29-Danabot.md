**Scenario:** The SOC team has detected suspicious activity in the network traffic, revealing that a machine has been compromised. Sensitive company information has been stolen. Your task is to use Network Capture (PCAP) files and Threat Intelligence to investigate the incident and determine how the breach occurred.

## Questions

> Q1: Which IP address was used by the attacker during the initial access?

![](assets/cyberdef/Pasted%20image%2020260530141055.png)

Before diving into the pcap file, I reviewed the conversation statistics. According to the statistics, there is a large amount of traffic from `188.114.97.3` , `195.133.88.98` , `142.250186.36` . After reviewing packets from these IP, there is not malicious network activity observed. 

![](assets/cyberdef/Pasted%20image%2020260530141224.png)

Next, I filtered DNS packet to observed any abnormal domain. There is one likely suspicious domain [portfolio.serveirc.com](#) which resolved to `62.173.142.148`

>✅ **Answer: 62.173.142.148**

---

> Q2: What is the name of the malicious file used for initial access?

![](assets/cyberdef/Pasted%20image%2020260530141307.png)

I filtered the suspicious IP and noticed a `GET` request from an internal IP to the malicious domain. It downloaded a file called `allegato_768.js`.

>✅ **Answer: allegato_768.js**

---

> Q3: What is the SHA-256 hash of the malicious file used for initial access?

![](assets/cyberdef/Pasted%20image%2020260530141353.png)

![](assets/cyberdef/Pasted%20image%2020260530141401.png)

I retrieved the malicious file by going into File > Export Objects > HTTP. It is normal for wireshark to shown login.php because wireshark names exported HTTP objects using the URL path from the HTTP request.

>✅ **Answer: 847b4ad90b1daba2d9117a8e05776f3f902dda593fb1252289538acf476c4268**

---

> Q4: Which process was used to execute the malicious file?

![](assets/cyberdef/Pasted%20image%2020260530141513.png)

Since, I have the file name and hash of the malicious file. I found [ANY.RUN](#) report related to this malicious file, `wscript.exe` was used to execute the malicious file.

>✅ **Answer: wscript.exe**

---

Q5: What is the file extension of the second malicious file utilized by the attacker?

![](assets/cyberdef/Pasted%20image%2020260530141547.png)

I de-obfuscated the `allegato_768.js` with [Deobfuscater](https://obf-io.deobfuscate.io/), and noticed that the malicious file downloaded another `resources.dll` from `sundata.top`.

>✅ **Answer: dll**

---

> Q6: What is the MD5 hash of the second malicious file?

![](assets/cyberdef/Pasted%20image%2020260530141729.png)

I used the same method to download the resources.dll and obtain the hash by running md5sum.

>✅ **Answer: e758e07113016aca55d9eda2b0ffeebe**
