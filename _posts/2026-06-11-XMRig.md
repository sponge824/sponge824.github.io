**Scenario:** During routine security audits at a startup, the SOC team detected unusual activity on Linux servers in the company’s infrastructure, including unexpected configuration changes and unfamiliar files in critical system directories. These anomalies suggest possible unauthorized access and raise concerns about the integrity of the server environment.

You received a disk image from one of the affected servers for forensic analysis. Your objective is to determine if a compromise has occurred, identify any tactics or tools used by a potential attacker, assess the scope and impact of the incident, and recommend mitigation strategies to safeguard against future breaches.

## Questions

> Q1: Assigning high-level privileges to a new user is essential in the attack chain, as it enables the attacker to execute commands with administrative access, ensuring persistent control over the system. What command did the attacker use to grant elevated privileges to the newly created user?

![](assets/cyberdef/Pasted%20image%2020260612152422.png)

![](assets/cyberdef/Pasted%20image%2020260611150001.png)

By analyzing the image file provided, there are 2 user account existed in the system. Inside user `ubuntu`, there is a file called `.bash_history`. It is used to records the command the user run in the bash shell. Analyzing the `.bash_history`file revealed that a new user `noah`was created assigned to sudo group.

>✅ **Answer: `sudo usermod -aG sudo noah`**

> Q2: Understanding the commands used by the attacker to cover their traces is essential for identifying attempts to hide malicious activity on the system. What is the second command the attacker used to erase evidence from the system?

Based on the `.bash_history`, we can observe that the attacker ran 2 `sudo rm` commands which remove the `.bash_history`and authentication log file `/var/log/auth.log`.

> Q8: Understanding how the attacker maintained elevated privileges without repeated permission prompts is essential for uncovering their methods of persistent access. What command did the attacker use to configure continuous privilege escalation without requiring repeated permission?

The attacker executed the command `echo 'Defaults !tty_tickets' >> /etc/sudoers`. The `sudoers` is the configuration file on Linux that used to limit who is allowed to use `sudo` and what commands they can run with elevated privilege. The attacker appended `Defaults !tty_tickets` which allowed `sudo`session to be reused across different terminal sessions without re-entering the password.

>✅ **Answer: `echo 'Defaults !tty_tickets' >> /etc/sudoers`**
