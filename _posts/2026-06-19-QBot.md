## Methodology

### C2 Communication
 
As the security team detected unusual network activity, our priority is to identify the C2 communication. By using netscan plugin and filtering for port 80, we managed to identify few suspicious IP.

```
94.140.112.73 - Latvia
196.204.5.11 - Egypt
45.147.230.104 - Germany
```

### Process Tree

![694](assets/cyberdef/Pasted%20image%2020260618212352.png)

The process `EXCEL.exe` (PID 4516) and `OneDrive.exe` (PID 7052) seems suspicious because there are a lot of child process `msedge.exe`spawned by them. The chronology would be a malware file was opened from Windows Explorer, and it's likely to be excel file extension`xlsx`, `xls` because `EXCEL.exe` was spawned.

### Locating Malicious File

![](assets/cyberdef/Pasted%20image%2020260619204904.png)

By using `windows.filescan`plugin, we can scan for file present in the memory image. We can then `grep` the Excel file extension (`xlsx`,`xls`) to identify the malicious file that was dropped on the host. We can see that there's a `Payment.xls`located at the Downloads folder of PC-28.

### File Extraction

![](assets/cyberdef/Pasted%20image%2020260619212933.png)

![](assets/cyberdef/Pasted%20image%2020260619213607.png)

We can extract the malicious file with `windows.dumpfiles` the flag `--virtaddr`which allow us to extract a specific file based on the virtual address. The virtual address can be obtained from the previous `windows.filescan`output `0xcd8ef507ac60`. Next, we calculate the file's hash which can be used for threat hunting to determine whether the same malicious file exists elsewhere in the system
